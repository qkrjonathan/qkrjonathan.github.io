The most basic guide to using Git CLI

8 September 2024

I hate Git documentation. I hate it. I just want to write something on this website and then push the updated files to the repo, and it never works the first time, and the documentation never works, and then I just bang my head on the wall until something works, and then I forget what makes it work, and then it's the same process all over again the next month.

This blog exists primarily to end that cycle, and, secondarily, for anyone else who might find it useful to have all this information in one place.

It's been an especially painful process since I've been used to Github Desktop for so long (I don't want to hear it) before I switched over to Linux and was subsequently forced to transition from GUI to command-line. But fear no more. This will be a definitive one-stop shop for all my CLI needs.

For context: My only practical use of GitHub has been to update my personal website (the one you are on now), and my writing blog, the25post[.]com — both of which are static GitHub Pages websites. This means pushing updated HTML, CSS, and JS files from my Zed clone to the actual repo.
